# 🤖 Gemini OCR 프로젝트 가이드 (AI 참고용)

이 프로젝트는 기존의 Tesseract OCR을 제거하고 **Gemini 1.5 Flash (또는 최신 Flash 모델)** API를 사용하여 보험 증권 분석을 수행하도록 개편되었습니다.

## 🛠 필수 설정 (개발 및 배포 시)
1. **환경 변수:** 서버 구동 환경에 `GEMINI_API_KEY`가 반드시 등록되어 있어야 합니다.
   - 로컬: `.env` 파일에 저장 (Git 업로드 금지 설정 완료)
   - 배포 서버: 서버 플랫폼(Vercel, Heroku 등)의 Environment Variables 설정에서 직접 입력
2. **모델 설정:** 현재 `gemini-2.5-flash-lite` 모델을 사용 중입니다. (할당량 및 속도 최적화 완료)
3. **이미지 처리:** 프론트엔드에서 이미지 비율을 분석하여 세로형(모바일)인 경우 상단 50%를 크롭하고, 가로형/정사각형인 경우 전체를 사용합니다. (가로 1024px 리사이징 및 JPEG 압축 포함)

## 🚀 배포 후 장애 조치 (Troubleshooting)
배포 후 "분석 실패" 또는 "404/401/429" 에러가 발생하면 다음을 확인하세요:
- **401/403:** `GEMINI_API_KEY`가 서버 설정에 누락되었거나 잘못 입력됨.
- **404:** 지정된 모델명이 해당 API 키 권한에서 사라짐 (서버 로그의 모델 목록 확인 필요).
- **429:** 무료 티어 할당량 초과 (분석 요청이 너무 잦음).

## 📝 주요 로직 설명
- `server.js`: `/api/ocr` 엔드포인트에서 이미지를 받아 Gemini API 호출.
- `index.html`: 이미지 리사이징, OCR 타이머(실시간 초 표시), 상품명 정제(무배당 등 제거) 로직 포함.
